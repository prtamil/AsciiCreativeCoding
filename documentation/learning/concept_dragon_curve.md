# Concept: Dragon Curve (L-System)

## Pass 1 — Understanding

### Core Idea
The dragon curve is a self-similar fractal generated by repeatedly folding a strip of paper in half in the same direction, then unfolding it so every fold is 90°. Equivalently, it's produced by an L-system string rewrite: at each step, insert R or L between every symbol. The string encodes a sequence of left/right turns.

### Mental Model
Take a strip of paper. Fold it in half right-to-left. Fold again, same direction. Unfold carefully so each crease is 90°. With 10 folds you see the dragon shape. The curve never crosses itself and tiles the plane when four copies are rotated.

### Key Equations
L-system production rules:
```
Axiom: FX
Rules: X → X+YF+
       Y → -FX-Y
```
Where F=forward, +=turn right 90°, -=turn left 90°, X/Y=no drawing.

Turn sequence shortcut: after n folds, turn i is:
```
turn_i = (((i & -i) << 1) & i) ? LEFT : RIGHT
```

### Data Structures
- String buffer or turn-sequence array
- Turtle graphics state: (x, y, angle)
- Drawing uses line segments → interpolated terminal characters

### Non-Obvious Decisions
- **Don't store the full string**: For order 20, the string has 2²⁰ ≈ 1M characters. Generate the turn sequence on-the-fly with the bit trick.
- **Bounding box first pass**: Walk the turtle without drawing to find min/max x,y, then scale and center.
- **Line drawing in terminal**: Use Bresenham's algorithm to draw segments, choose `|`, `-`, `/`, `\` by slope angle.
- **Color by iteration**: Color each segment by its position in the sequence (hue cycling) to reveal the recursive structure.

### Key Constants
| Name | Role |
|------|------|
| ORDER | recursion depth (10–15 is visible, 20 is maximum) |
| STEP | pixel length of each segment |
| TURN_ANGLE | 90° for dragon curve |

### Open Questions
- How does the dragon curve relate to the Heighway dragon?
- Try TURN_ANGLE = 60° — what shape emerges?
- Four copies of the dragon curve rotated by 0°,90°,180°,270° — do they tile?

## Pass 2 — Implementation

### Pseudocode
```
# Bit-trick turn sequence: for step i (1-indexed)
# find the highest power of 2 that divides i,
# then check next bit
turn(i) → LEFT or RIGHT:
    k = i & (-i)      # lowest set bit = highest 2^n dividing i
    return (i & (k<<1)) ? LEFT : RIGHT

generate_path(ORDER):
    n_steps = 2^ORDER
    (x,y) = (0,0); angle = 0
    path = []
    for i in 1..n_steps:
        if turn(i) == RIGHT: angle -= 90°
        else:                angle += 90°
        dx = round(cos(angle)); dy = round(sin(angle))
        path.append(line(x,y, x+dx,y+dy))
        x+=dx; y+=dy
    return path

bounding_box(path) → (minx,miny,maxx,maxy)

render(path, scale, offset):
    for each segment in path:
        draw_line(segment.x1*scale+offx, segment.y1*scale+offy,
                  segment.x2*scale+offx, segment.y2*scale+offy)
```

### Module Map
```
§1 config    — ORDER, initial scale
§2 generate  — turn(), generate_path()
§3 fit       — bounding_box(), compute scale to fit terminal
§4 draw      — draw_line() with slope-appropriate chars
§5 app       — main loop, keys (order up/down, zoom, color mode)
```

### Data Flow
```
ORDER → turn sequence (bit trick) → turtle path
→ bounding box → scale/center transform → draw line segments
```
